Implementation Spec — USC Attendance System

Check-Out Flow, Time-Window Enforcement & Schedule Management

Adding checkout capability (morning check-in / afternoon check-out), duty time-window enforcement via OrgProfile, and a super-admin schedule management UI. Event assignments are explicitly OUT of scope.

01Schema Changes

1a. Add checkOutAt to Attendance done
Add nullable DateTime? checkOutAt field to the Attendance model. This stores when the officer checked out. Null means still open (checked in but not yet checked out).
prisma/schema.prisma
FieldTypeNotes
checkOutAtDateTime?NEW — null = open, set on checkout
1b. Replace @@unique with partial unique indexes done
Remove the Prisma-level @@unique([eventId, userId]) and @@unique([userId, dutyDate]) constraints. Replace with PostgreSQL partial unique indexes that only enforce uniqueness when checkOutAt IS NULL (i.e., only one open record per officer per event/day). This allows unlimited historical rows while preventing race conditions.
prisma/schema.prismaprisma/migrations/…/migration.sql (raw SQL block)
-- Added manually to the generated migration.sql CREATE UNIQUE INDEX attendance_event_open_unique ON "Attendance" ("eventId", "userId") WHERE "checkOutAt" IS NULL AND "eventId" IS NOT NULL; CREATE UNIQUE INDEX attendance_duty_open_unique ON "Attendance" ("userId", "dutyDate") WHERE "checkOutAt" IS NULL AND "dutyDate" IS NOT NULL;
1c. Create OrgProfile model done
Singleton model to store org-wide settings. Fields for duty time-window enforcement. Only one row ever exists.
prisma/schema.prisma
FieldTypeDefaultPurpose
idString (PK)uuidSingleton ID
dutyCheckInCutoffString"12:00"HH:mm — duty check-ins must occur before this
dutyCheckOutStartString"12:00"HH:mm — duty check-outs must occur at or after this

04API Endpoints

4a. Create checkout endpoint done
POST /api/attendance/checkout — Body: { eventId?, dutyDate?, userId }
• Self-checkout: authUser.id === body.userId — no role restriction
• Manual checkout: authUser.id !== body.userId — require staff/super_admin
• Self-targeting guard: staff cannot manually check themselves out (403)
• Find open record (checkOutAt IS NULL), 404 if none
• Duty time-window: fetch OrgProfile, compare server time (Asia/Manila) vs dutyCheckOutStart
• Set checkOutAt = new Date(), save, return
api/attendance/checkout.ts
4b. Update attendance check-in (index.ts) done
Replace findUnique duplicate check with findFirst where checkOutAt IS NULL. Add try/catch for Prisma P2002 unique constraint violation → same 409 message.
api/attendance/index.ts
4c. Update duty check-in + time-window enforcement done
Same duplicate-check pattern as 4b. Add time-window enforcement: fetch OrgProfile, compare current server time (Asia/Manila) against dutyCheckInCutoff. Reject 403 if currentTime >= cutoff. Create default OrgProfile if none exists.
api/attendance/duty.ts
4d. Update CSV export done
Add "Checked Out At" column to CSV header and row data.
api/attendance/export.ts
